### 主方向：Research

#### 1. 选择理由

我对技术开发和研究分析都比较感兴趣，但目前我的技术基础还不足以独立完成较复杂的开发任务。相比直接从 Dev 方向开始，我认为自己现阶段更适合先从研究切入，通过资料收集、产品分析和用户场景拆解，逐渐建立对 Monad 生态和链上产品的理解。

同时，我并不希望完全放弃技术方向。在研究过程中，我也会继续尝试简单的链上交互、数据整理或 Demo 验证，让研究结论尽量与真实产品和技术场景结合。

#### 2. 希望服务的问题

我希望研究的问题是：

**Monad 上什么类型的产品或交互，能够让普通用户愿意持续使用，而不仅仅是为了完成一次空投任务或测试网交互？**

目前许多链上产品对新用户来说门槛较高，交互过程也比较工具化。用户可能完成一次转账、签到或任务后就离开，缺少长期使用的动力。

我希望从用户需求、交互频率、反馈机制和产品场景等方面，分析什么样的链上应用更容易形成持续使用习惯。

#### 3. 本周最小产出

本周计划完成一份轻量级研究笔记，内容包括：

* 选取 3—5 个 Monad 生态或其他公链上的高频交互产品。
* 分析它们的目标用户、核心功能和使用流程。
* 总结这些产品如何吸引用户首次使用，以及如何鼓励用户持续返回。
* 提出 1—2 个可能适合 Monad 的产品方向或交互机制。
* 如果时间允许，使用简单流程图或页面草图展示其中一个方向。

最终产出形式可以是一份 GitHub Markdown 研究报告，并继续放在当前的 `monad-builder-camp` 仓库中。

#### 4. 参考资料

本周计划参考以下资料：

* Monad 官方文档及生态项目介绍。
* Monad 测试网中的实际交互体验。
* 不同公链上的任务平台、链游、社交产品和用户激励案例。
* DappRadar、DefiLlama 等平台中的产品数据和项目分类。
* 项目官网、白皮书、X 账号及用户反馈。

在使用 AI 帮助整理资料和搭建分析框架的同时，我会人工检查项目功能、用户场景和结论是否合理。

#### 5. Week 3 希望承担的角色

Week 3 我希望继续承担 Research 相关角色，例如：

* 搜集和整理项目资料。
* 分析目标用户和使用场景。
* 对竞品功能和交互流程进行对比。
* 将研究结论整理成产品需求或功能建议。
* 与 Dev 成员合作，帮助确定最值得验证的最小功能。

如果团队需要，我也愿意参与简单的测试、链上交互验证、README 整理或 Demo 展示，从研究和产品角度支持开发工作。
